# UXレビューチェックリスト

UI実装時・レビュー時に確認すべき項目。

---

## 基本チェック

### 視覚デザイン

- [ ] **一貫性**: デザインシステムに従っているか
- [ ] **視覚的階層**: 重要な要素が目立っているか
- [ ] **余白**: 適切な間隔が確保されているか
- [ ] **コントラスト**: テキストが読みやすいか（WCAG準拠）
- [ ] **レスポンシブ**: モバイル/タブレット/デスクトップで適切か

**関連原則**: Aesthetic-Usability Effect, Visual Hierarchy

---

### フィードバック

- [ ] **即時性**: 0.4秒以内に何らかの反応があるか
- [ ] **ローディング**: 処理中であることが分かるか
- [ ] **進捗表示**: 長い処理には進捗が表示されるか
- [ ] **成功確認**: 操作の成功が明確に伝わるか
- [ ] **エラー表示**: エラーの原因と対処法が分かるか

**関連原則**: Doherty Threshold, Labor Illusion, Peak-End Rule

---

### フォーム

- [ ] **デフォルト値**: 適切なデフォルト値が設定されているか
- [ ] **バリデーション**: リアルタイムでエラーが表示されるか
- [ ] **エラーメッセージ**: 具体的で分かりやすいか
- [ ] **必須表示**: 必須/任意が明確か
- [ ] **ステップ分割**: 長いフォームは分割されているか
- [ ] **入力補助**: オートコンプリート等が活用されているか

**関連原則**: Default Bias, Cognitive Load, Progressive Disclosure

---

### ナビゲーション

- [ ] **現在位置**: 今どこにいるか分かるか
- [ ] **パンくずリスト**: 階層が深い場合に表示されているか
- [ ] **戻る操作**: 前の画面に戻れるか
- [ ] **標準パターン**: 慣習的なUIパターンを使用しているか

**関連原則**: Familiarity Bias, Cognitive Load

---

## シナリオ別チェック

### 初回訪問ユーザー

- [ ] **価値提案**: 何ができるかが3秒で分かるか
- [ ] **オンボーディング**: 必要に応じてガイドがあるか
- [ ] **最初の成功体験**: すぐに価値を体験できるか
- [ ] **ヘルプ**: 困った時に助けを得られるか

**関連原則**: Familiarity Bias, Progressive Disclosure, Peak-End Rule

---

### 待機中ユーザー

- [ ] **即時反応**: クリック/タップに即座に反応するか
- [ ] **処理表示**: 何が行われているか分かるか
- [ ] **キャンセル可能**: 長い処理はキャンセルできるか
- [ ] **バックグラウンド**: 待機中に他の操作ができるか

**関連原則**: Doherty Threshold, Labor Illusion

---

### 選択・購入ユーザー

- [ ] **比較しやすさ**: 選択肢を比較できるか
- [ ] **推奨表示**: おすすめが明示されているか
- [ ] **安心感**: 不安を解消する情報があるか（保証、レビュー等）
- [ ] **緊急性**: 過度な煽りになっていないか

**関連原則**: Decoy Effect, Social Proof, Scarcity Effect

---

### 離脱しそうなユーザー

- [ ] **進捗保存**: 途中経過が保存されているか
- [ ] **リマインド**: 未完了タスクの通知があるか
- [ ] **ゴール提示**: あとどれくらいか分かるか
- [ ] **インセンティブ**: 完了のメリットが伝わるか

**関連原則**: Zeigarnik Effect, Goal Gradient Effect, Sunk Cost Effect

---

## ダークパターン回避

### 確認すべき項目

- [ ] **Confirmshaming**: 拒否オプションに恥ずかしい文言を使っていないか
- [ ] **Hidden Costs**: 追加料金を最後まで隠していないか
- [ ] **Roach Motel**: 登録は簡単で解約が困難になっていないか
- [ ] **Trick Questions**: チェックボックスの文言が紛らわしくないか
- [ ] **Forced Continuity**: 自動更新が分かりにくくないか
- [ ] **Friend Spam**: 連絡先を無断でアクセスしていないか
- [ ] **Disguised Ads**: 広告がコンテンツに見せかけられていないか
- [ ] **Bait and Switch**: 約束した内容と異なるものを提供していないか

---

## アクセシビリティ

### 基本

- [ ] **キーボード操作**: 全ての操作がキーボードでできるか
- [ ] **フォーカス表示**: フォーカス位置が視覚的に分かるか
- [ ] **Alt属性**: 画像にalt属性があるか
- [ ] **見出し構造**: 適切な見出しレベルが使われているか
- [ ] **ラベル**: フォーム要素に適切なラベルがあるか

### 色・コントラスト

- [ ] **色だけに頼らない**: 色以外でも情報が伝わるか
- [ ] **コントラスト比**: WCAG AA基準（4.5:1）を満たすか
- [ ] **色覚多様性**: 色覚特性に配慮されているか

---

## パフォーマンス

### 体感速度

- [ ] **初期表示**: ファーストビューが3秒以内に表示されるか
- [ ] **インタラクション**: クリックから0.4秒以内に反応するか
- [ ] **スクロール**: スムーズにスクロールするか
- [ ] **アニメーション**: 60fpsで動作するか

### 最適化

- [ ] **画像**: 適切なサイズ・形式で配信されているか
- [ ] **遅延読み込み**: 非表示部分は遅延読み込みされているか
- [ ] **キャッシュ**: 適切にキャッシュされているか

---

## コンポーネント別チェック

### ボタン

- [ ] 押せることが分かる見た目か
- [ ] hover/active/disabled状態が明確か
- [ ] 十分なタップ領域があるか（44px以上）
- [ ] ラベルが行動を示しているか

### フォーム入力

- [ ] プレースホルダーがラベル代わりになっていないか
- [ ] 入力形式のヒントがあるか
- [ ] エラー時に入力値が消えないか
- [ ] パスワード表示/非表示の切り替えがあるか

### モーダル/ダイアログ

- [ ] 閉じる方法が明確か（×ボタン、オーバーレイクリック）
- [ ] ESCキーで閉じられるか
- [ ] フォーカストラップが実装されているか
- [ ] 背景がスクロールしないか

### トースト/通知

- [ ] 表示時間は適切か（成功: 3秒、エラー: 5秒以上）
- [ ] 手動で閉じられるか
- [ ] スクリーンリーダーに読み上げられるか
- [ ] 複数通知が重ならないか

### テーブル/リスト

- [ ] 空状態のメッセージがあるか
- [ ] ソート/フィルター状態が分かるか
- [ ] 長いリストはページング/無限スクロールがあるか
- [ ] 行のhover状態があるか

---

## クイックチェック

UI実装の最終確認用。

```
□ 見た目が美しいか（Aesthetic-Usability Effect）
□ 0.4秒以内に反応するか（Doherty Threshold）
□ 既知のパターンを使っているか（Familiarity Bias）
□ 処理中が見えるか（Labor Illusion）
□ 情報が段階的か（Progressive Disclosure）
□ 重要なものが目立つか（Visual Hierarchy）
□ デフォルト値が適切か（Default Bias）
□ エラーが分かりやすいか（Cognitive Load）
□ 終わりが印象的か（Peak-End Rule）
□ ダークパターンを使っていないか
```
